## 前提条件

### 1. 服务端准备

在创建快捷指令之前，请确保以下服务已部署并正常运行：

- **Knowledge Collector API 服务器**：已部署并可通过公网或局域网访问
- **思源笔记**：已启动并配置好 API Token
- **服务器 URL**：记录下你的 API 端点地址，格式如下：

```
https://your-server.com/api/collect
```

> [!IMPORTANT]
> 如果你的服务部署在本地网络中，请确保你的 iPhone 与服务器在同一网络下，
> 或者通过内网穿透（如 frp、Cloudflare Tunnel）将服务暴露到公网。

### 2. 设备要求

- iPhone 或 iPad，运行 iOS 15.0 或更高版本
- 已安装「快捷指令」App（系统自带）
- 已安装微信

### 3. 获取配置信息

你需要准备以下信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| API 端点 | 收藏服务的接口地址 | `https://your-server.com/api/collect` |
| API 密钥 | 用于身份验证的密钥 | `your-api-key` |
| 默认来源 | 内容来源标识 | `wechat` |

## 快速开始

如果你熟悉 iOS 快捷指令，以下是核心工作流概览：

```
接收分享内容 → 提取 URL → 可选添加标签 → POST 到 API → 显示通知
```

---

## 详细创建步骤

### 第 1 步：打开快捷指令 App

1. 在 iPhone 主屏幕找到并打开「**快捷指令**」App
2. 点击右上角的 **「+」** 按钮，创建新的快捷指令
3. 点击顶部的快捷指令名称，将其命名为 **「收藏到知识库」**

### 第 2 步：配置接收分享输入

> 📸 操作位置：快捷指令编辑器顶部

1. 点击顶部的 **「ⓘ」** 图标（或快捷指令名称旁的下拉箭头）
2. 选择 **「在共享表单中显示」**（Share Sheet）
3. 在 **「接收内容类型」** 中，勾选以下选项：
   - ✅ **URL**
   - ✅ **文本**
   - ✅ **Safari 网页**
   - ✅ **文章**
4. 点击 **「完成」**

> [!TIP]
> 同时选择 URL 和文本类型，可以兼容微信分享的不同格式。
> 某些文章可能以纯文本形式分享链接。

### 第 3 步：添加「获取快捷指令输入的 URL」操作

> 📸 操作位置：操作搜索栏

1. 点击 **「添加操作」**
2. 在搜索栏搜索 **「URL」**
3. 选择 **「从输入获取 URL」**（Get URLs from Input）
4. 确保输入来源设置为 **「快捷指令输入」**

**此步骤的作用**：无论用户分享的是纯 URL 还是包含 URL 的文本，都能正确提取出链接地址。

### 第 4 步：（可选）添加标签输入提示

> 📸 操作位置：添加操作 → 脚本

1. 点击 **「添加操作」**
2. 搜索 **「要求输入」**（Ask for Input）
3. 配置如下：
   - **提示文字**：`请输入标签（用逗号分隔，可留空）`
   - **输入类型**：选择 **「文本」**
   - **默认值**：留空
4. 将此操作的输出变量重命名为 **`userTags`**

> [!NOTE]
> 如果你不需要每次都手动添加标签，可以跳过此步骤。
> 可在配置文件中设置 `show_tag_prompt: false` 来禁用标签提示。

### 第 5 步：添加「获取 URL 内容」操作（核心步骤）

> 📸 操作位置：添加操作 → 网络

1. 点击 **「添加操作」**
2. 搜索 **「获取 URL 内容」**（Get Contents of URL）
3. 配置请求参数：

   **基本设置：**
   - **URL**：填入你的 API 端点地址
     ```
     https://your-server.com/api/collect
     ```
   - **方法**：选择 **`POST`**

   **请求头（Headers）：**

   点击 **「添加新头部」**，添加以下头部：

   | 键（Key） | 值（Value） |
   |-----------|------------|
   | `Content-Type` | `application/json` |
   | `Authorization` | `Bearer your-api-key` |

   **请求体（Request Body）：**

   选择 **「JSON」** 格式，然后添加以下字段：

   | 键（Key） | 类型 | 值（Value） |
   |-----------|------|------------|
   | `url` | 文本 | *选择第 3 步的输出变量（URLs）* |
   | `source` | 文本 | `wechat` |
   | `tags` | 文本 | *选择第 4 步的 `userTags` 变量*（如有） |

   **完整的 JSON 请求体示例：**
   ```json
   {
     "url": "https://mp.weixin.qq.com/s/xxxxx",
     "source": "wechat",
     "tags": ["AI", "技术"]
   }
   ```

> [!WARNING]
> 请务必将 `your-server.com` 和 `your-api-key` 替换为你的实际服务器地址和 API 密钥。
> 切勿在公共场合或截图中泄露你的 API 密钥。

### 第 6 步：添加结果通知

> 📸 操作位置：添加操作 → 脚本

1. 点击 **「添加操作」**
2. 搜索 **「显示通知」**（Show Notification）
3. 配置如下：
   - **标题**：`知识库收藏`
   - **正文**：`✅ 文章已成功保存到知识库！`

#### 可选：添加错误处理

为了获得更好的使用体验，建议添加错误处理：

1. 在「获取 URL 内容」操作后，添加 **「如果」**（If）操作
2. 条件设置为：检查返回的字典中 `success` 字段 **等于** `true`
3. **如果是**：显示通知 `✅ 文章已成功保存！`
4. **否则**：显示通知 `❌ 保存失败，请检查网络或服务器状态`

### 第 7 步：保存快捷指令

1. 点击右上角的 **「完成」** 按钮
2. 快捷指令将自动保存

## 完整工作流一览

```
┌─────────────────────────────────────────────┐
│  1. 接收分享输入（URLs / 文本）               │
│     ↓                                       │
│  2. 从输入中获取 URL                          │
│     ↓                                       │
│  3. [可选] 弹窗询问标签                       │
│     ↓                                       │
│  4. 发送 POST 请求到 API                      │
│     • URL: https://your-server.com/api/collect│
│     • Method: POST                           │
│     • Headers: Content-Type, Authorization   │
│     • Body: { url, source, tags }            │
│     ↓                                       │
│  5. 显示结果通知                              │
│     • 成功 → ✅ 已保存                       │
│     • 失败 → ❌ 保存失败                     │
└─────────────────────────────────────────────┘
```

## 配置微信分享面板

### 将快捷指令添加到微信的分享菜单中

1. 打开微信，进入任意一篇公众号文章
2. 点击右上角的 **「···」** 更多按钮
3. 在底部分享面板中，向右滑动找到 **「更多」**
4. 如果看不到快捷指令选项，请按以下步骤操作：

#### 启用快捷指令分享扩展

1. 打开 **「设置」** → **「快捷指令」**
2. 确保 **「允许不受信任的快捷指令」**（或「专用共享」）已开启
3. 返回微信分享面板
4. 点击 **「更多」** → **「编辑」**
5. 找到 **「收藏到知识库」** 快捷指令，将其开关打开
6. 可以拖动调整其位置，放到前排方便使用

> [!TIP]
> 建议将「收藏到知识库」放在分享面板的第一行，
> 这样每次使用时可以更快速地找到它。

### 使用方法

1. 在微信中打开一篇公众号文章
2. 点击右上角 **「···」**
3. 在分享面板中选择 **「收藏到知识库」**
4. （如果启用了标签提示）输入标签，或直接留空点击「完成」
5. 等待几秒，收到通知即表示保存成功 🎉

## 故障排除

### 常见问题及解决方案

#### ❌ 问题 1：快捷指令在分享面板中不显示

**可能原因：**
- 未启用「在共享表单中显示」选项

**解决方案：**
1. 打开「快捷指令」App
2. 长按「收藏到知识库」快捷指令 → 选择「详细信息」
3. 确认「在共享表单中显示」已开启
4. 检查接收类型是否包含 URL 和文本

---

#### ❌ 问题 2：提示「不允许运行不受信任的快捷指令」

**解决方案：**
1. 前往「设置」→「快捷指令」
2. 开启「允许不受信任的快捷指令」
3. 如果选项灰色不可用，先随意运行系统内置的一个快捷指令，然后再试

---

#### ❌ 问题 3：请求超时或无法连接服务器

**可能原因：**
- 服务器未启动
- 网络不可达（服务器在内网，手机在外网）
- URL 地址填写错误

**解决方案：**
1. 确认服务器正在运行（使用 `test-shortcut.sh` 脚本测试）
2. 检查手机与服务器的网络连通性
3. 如果服务器在内网，配置内网穿透或 VPN
4. 检查 URL 是否以 `http://` 或 `https://` 开头

---

#### ❌ 问题 4：返回 401 未授权错误

**解决方案：**
1. 检查 `Authorization` 头部的 API 密钥是否正确
2. 确认格式为 `Bearer your-api-key`（注意 Bearer 后有一个空格）
3. 在服务器端确认密钥未过期

---

#### ❌ 问题 5：文章 URL 无法正确提取

**可能原因：**
- 微信分享的内容格式不一致

**解决方案：**
1. 确保快捷指令中包含「从输入获取 URL」操作
2. 同时接受 URL 和文本类型的输入
3. 如果仍有问题，尝试在微信中选择「复制链接」后手动运行快捷指令

---

#### ❌ 问题 6：通知不显示

**解决方案：**
1. 前往「设置」→「通知」→「快捷指令」
2. 确保通知权限已开启
3. 检查是否开启了「专注模式」导致通知被屏蔽

## 高级配置

### 自定义默认标签

如果你希望自动为所有微信文章添加特定标签，可以在快捷指令的 JSON body 中硬编码：

```json
{
  "url": "<提取的 URL>",
  "source": "wechat",
  "tags": ["微信", "待阅读"]
}
```

### 多端点支持

如果你有多个知识库系统，可以创建多个快捷指令变体，或使用「从菜单中选取」操作让用户选择目标知识库：

1. 添加「从菜单中选取」操作
2. 选项设置为：`思源笔记`、`Notion`、`其他`
3. 根据选择使用不同的 API 端点发送请求

### 配置文件

参考 `shortcut-config.json` 文件了解所有可配置项。
